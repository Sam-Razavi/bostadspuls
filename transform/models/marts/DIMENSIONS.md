# Dimensional Model Design

## Overview

The `bostadspuls_marts` dataset uses a star schema centred on `fact_sales`.
All dimension tables use surrogate keys generated with `dbt_utils.generate_surrogate_key`.

## Dimensions

### `dim_location`

**Grain:** one row per unique (county, municipality, city) combination.
**Key column:** `location_key` (surrogate).
**Join to SCB:** via `region_code` (ISO 3166-2:SE).

### `dim_property_type`

**Grain:** one row per distinct Booli `object_type` (lowercase Swedish).
**Key column:** `property_type_key` (surrogate).
**Extras:** English translation (`object_type_en`), `is_apartment` boolean.

### `dim_date`

**Grain:** one row per calendar date, 2020-01-01 to 2030-12-31.
**Generated by:** `dbt_date.get_date_dimension` — provides year, month, quarter,
day-of-week, and other calendar attributes out of the box.

## Fact table

### `fact_sales`

**Grain:** one row per sold property (one Booli listing).
**Measures:** `sold_price_sek`, `list_price_sek`, `living_area_sqm`, `rooms`,
`price_per_sqm`, `price_vs_list_pct`.
**Foreign keys:** `location_key`, `property_type_key`, `sold_date_key`.
